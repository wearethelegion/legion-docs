legion integruje się z przepływem pracy GitLab poprzez potok GitLab CI/CD lub z GitLab Duo.

W obu przypadkach legion będzie modułem GitLab.

---

## GitLab CI

legion działa w zwykłym potoku GitLab. Możesz wbudować go w potok jako [komponent CI](https://docs.gitlab.com/ee/ci/components/)

Tutaj obowiązkowego przez społeczność komponentu CI/CD dla legion — [nagyv/gitlab-legion](https://gitlab.com/nagyv/gitlab-legion).

---

### Cechy

- **Użyj konfiguracji dla każdego zadania**: Skonfiguruj legion z niestandardowym katalogiem konfiguracyjnym, na przykładzie `./config/#custom-directory`, aby włączyć lub funkcjonalności na każde wywołanie legion.
- **Konfiguracja minimalna**: Komponent CI konfiguruje legion w tle, wystarczy skonfigurować legion i początkowy monit.
- **Elastyczny**: Komponent CI obsługuje kilka danych, które są stosowane w celu dostosowania jego zachowania

---

### Organizować coś

1. Przechowuj dane uwierzytelniające legion w formacie JSON jako zmienne parametry CI typu pliku w specyfikacji **Ustawienia** > **CI/CD** > **Zmienne**. Zachowaj, aby być oznaczony jako „Zamaskowane i ukryte”.
2. Dodaj elementy do swojego pliku `.gitlab-ci.yml`.

   ```yaml title=".gitlab-ci.yml"
   include:
     - component: $CI_SERVER_FQDN/nagyv/gitlab-legion/legion@2
       inputs:
         config_dir: ${CI_PROJECT_DIR}/legion-config
         auth_json: $LEGION_AUTH_JSON # The variable name for your Legion authentication JSON
         command: optional-custom-command
         message: "Your prompt here"
   ```

Więcej danych dotyczących użycia [sprawdź opis](https://gitlab.com/explore/catalog/nagyv/gitlab-legion) tego komponentu.

## Duet GitLab

legion integruje się z przepływem pracy GitLab.
Wspomnij o `@legion` w komentarzu, a legion wykonaj zadania w ramach swojego potoku GitLab CI.

---

### Cechy

- **Problem związany z segregacją**: Poproś legion o szczegółowe wyjaśnienie problemu i wyjaśnienie go.
- **Napraw i zaimplementuj**: Poproś legion o naprawienie problemu lub zaimplementowanie funkcji.
  Utworzył nowy oddział i zgłosi ataku ze sobą.
- **Bezpieczny**: legion działa na modułach GitLab.

---

### Organizować coś

legion działa w potoku CI/CD GitLab. Oto, czego potrzebujesz, aby przejść do:

:::tip
Aktualne instrukcje dotyczące [**dokumentacji GitLab**](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/).
:::

1.  Skonfiguruj swoje środowisko GitLab
2.  Skonfiguruj CI/CD
3.  Uzyskaj klucz API dostawca modelu AI
4.  Utwórz konto usługi
5.  Skonfiguruj zmienne CI/CD
6.  Utwórz plik konfiguracyjny, oto przykład:

    image: node:22-slim

      - git config --global user.email "legion@gitlab.com"
      - git config --global user.name "Legion"
      - echo "Testing glab"
      - glab issue list
      - echo "Running Legion"
      - |
        legion run "
        You are an AI assistant helping with GitLab operations.

        Context: $AI_FLOW_CONTEXT
        Task: $AI_FLOW_INPUT
        Event: $AI_FLOW_EVENT

        Please execute the requested task using the available GitLab tools.
        Be thorough in your analysis and provide clear explanations.

        <important>

### Przykłady

Oto kilka możliwości wykorzystania legion w GitLabie.

:::tip
Można zastosować inną metodę ograniczającą niż `@legion`.
:::

- **Wyjaśnij problem**

  Dodaj dziesięć komentarzy w numerze GitLab.

  ```
  @legion explain this issue
  ```

  legion przeczytaj problem i odpowiedź z wyjaśnieniem.

- **Napraw problem**

  W numerze GitLab powiedz:

  ```
  @legion fix this
  ```

  legion utworzy nową podstawę, zaimplementuje zmiany i spowoduje, że skalowanie będzie możliwe.

- **Przejrzyj prośby o połączenie**

  Zostaw komentarz w sprawie rozwiązania połączenia GitLab.

  ```
  @legion review this merge request
  ```

  legion sprawdzenie rozwiązania i przepuszczenie.
