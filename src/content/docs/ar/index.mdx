[**Legion**](/) هو وكيل ترميز بالذكاء الاصطناعي مفتوح المصدر. يتوفر كواجهة terminal، وتطبيق لسطح المكتب، أو إضافة لبيئة تطوير متكاملة (IDE).

![واجهة Legion في terminal مع سمة legion](../../../assets/lander/screenshot.png)

لنبدأ.

---

#### المتطلبات

لاستخدام Legion في terminal، ستحتاج إلى:

1. محاكي terminal حديث مثل:
   - [WezTerm](https://wezterm.org) متعدد المنصات
   - [Alacritty](https://alacritty.org) متعدد المنصات
   - [Ghostty](https://ghostty.org) على Linux وmacOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/) على Linux وmacOS

2. مفاتيح API لمزوّدي نماذج اللغة (LLM) الذين تريد استخدامهم.

---

## التثبيت

أسهل طريقة لتثبيت Legion هي عبر سكربت التثبيت.

```bash
curl -fsSL https://wearethelegion.com/install | bash
```

يمكنك أيضا تثبيته عبر الأوامر التالية:

- **باستخدام Node.js**

        <Tabs>

      <TabItem label="npm">
      ```bash
      npm install -g @wearethelegion/legion
      ```

          </TabItem>

        <TabItem label="Bun">
        ```bash
        bun install -g @wearethelegion/legion
        ```

          </TabItem>

        <TabItem label="pnpm">
        ```bash
        pnpm install -g @wearethelegion/legion
        ```

          </TabItem>

        <TabItem label="Yarn">
        ```bash
        yarn global add wearethelegion
        ```

      </TabItem>

  </Tabs>

- **باستخدام Homebrew على macOS وLinux**

  ```bash
  brew install wearethelegion/tap/legion
  ```

  > نوصي باستخدام tap الخاص بـ Legion للحصول على أحدث الإصدارات. صيغة `brew install legion` الرسمية تُدار بواسطة فريق Homebrew ويتم تحديثها بوتيرة أقل.

- **باستخدام Paru على Arch Linux**

  ```bash
  sudo pacman -S legion           # Arch Linux (Stable)
  paru -S legion-bin              # Arch Linux (Latest from AUR)
  ```

#### Windows

:::tip[موصى به: استخدم WSL]
لأفضل تجربة على Windows، نوصي باستخدام [Windows Subsystem for Linux (WSL)](/docs/windows-wsl). يوفر أداء أفضل وتوافقا كاملا مع ميزات Legion.
:::

- **باستخدام Chocolatey**

  ```bash
  choco install legion
  ```

- **باستخدام Scoop**

  ```bash
  scoop install legion
  ```

- **باستخدام NPM**

  ```bash
  npm install -g @wearethelegion/legion
  ```

- **باستخدام Mise**

  ```bash
  mise use -g github:wearethelegion/legion
  ```

- **باستخدام Docker**

  ```bash
  docker run -it --rm ghcr.io/wearethelegion/legion
  ```

دعم تثبيت Legion على Windows باستخدام Bun قيد العمل حاليا.

يمكنك أيضا تنزيل الملف التنفيذي من صفحة [Releases](https://github.com/wearethelegion/legion/releases).

## الإعداد

مع Legion يمكنك استخدام أي مزود LLM عبر إعداد مفاتيح API الخاصة به.

إذا كنت جديدا على استخدام مزوّدي LLM، فنوصي باستخدام [Legion Zen](/docs/zen).
إنها قائمة منتقاة من النماذج تم اختبارها والتحقق منها بواسطة فريق Legion.

1. شغّل الأمر `/connect` في واجهة TUI، واختر legion، ثم انتقل إلى [wearethelegion.com/auth](https://wearethelegion.com/auth).

   ```txt
   /connect
   ```

2. سجّل الدخول، وأضف تفاصيل الدفع، ثم انسخ مفتاح API.

3. الصق مفتاح API.

   ```txt
   ┌ API key
   │
   │
   └ enter
   ```

بدلا من ذلك يمكنك اختيار أحد المزوّدين الآخرين. [اعرف المزيد](/docs/providers#directory).

## التهيئة

بعد أن قمت بإعداد مزوّد، يمكنك الانتقال إلى المشروع الذي تريد العمل عليه.

```bash
cd /path/to/project
```

ثم شغّل Legion.

```bash
legion
```

بعد ذلك، هيّئ Legion للمشروع عبر تشغيل الأمر التالي.

```bash frame="none"
/init
```

سيجعل هذا Legion يحلل مشروعك ويُنشئ ملف `AGENTS.md` في جذر المشروع.

:::tip
يستحسن أن تقوم بعمل commit لملف `AGENTS.md` الخاص بمشروعك إلى Git.
:::

يساعد هذا Legion على فهم بنية المشروع وأنماط الترميز المستخدمة.

## الاستخدام

أصبحت الآن جاهزا لاستخدام Legion للعمل على مشروعك. لا تتردد في سؤاله أي شيء!

إذا كنت جديدا على استخدام وكيل ترميز بالذكاء الاصطناعي، فإليك بعض الأمثلة التي قد تساعد.

---

### اطرح أسئلة

يمكنك أن تطلب من Legion شرح قاعدة الشيفرة لك.

:::tip
استخدم المفتاح `@` للبحث التقريبي عن الملفات داخل المشروع.
:::

```txt frame="none" "@packages/functions/src/api/index.ts"
How is authentication handled in @packages/functions/src/api/index.ts
```

يفيد هذا عندما تكون هناك أجزاء من قاعدة الشيفرة لم تعمل عليها.

---

### أضف ميزات

يمكنك أن تطلب من Legion إضافة ميزات جديدة إلى مشروعك، لكننا نوصي أولا بطلب إنشاء خطة.

1. **أنشئ خطة**

   لدى Legion _وضع Plan_ يعطل قدرته على إجراء التغييرات، ويقترح بدلا من ذلك _كيف_ سينفّذ الميزة.

   انتقل إليه باستخدام مفتاح **Tab**. سترى مؤشرا لذلك في الزاوية السفلية اليمنى.

   ```bash frame="none" title="التبديل إلى وضع Plan"
   <TAB>
   ```

   الآن لنصف ما نريده أن يفعله.

   ```txt frame="none"
   When a user deletes a note, we'd like to flag it as deleted in the database.
   Then create a screen that shows all the recently deleted notes.
   From this screen, the user can undelete a note or permanently delete it.
   ```

   احرص على تزويد Legion بتفاصيل كافية ليفهم ما تريد. يساعد أن تتحدث إليه كما لو كنت تتحدث إلى مطور مبتدئ ضمن فريقك.

   :::tip
   أعطِ Legion سياقا وأمثلة كافية لمساعدته على فهم ما تريد.
   :::

2. **حسّن الخطة**

   بعد أن يقدم لك خطة، يمكنك إعطاؤه ملاحظات أو إضافة تفاصيل أكثر.

   ```txt frame="none"
   We'd like to design this new screen using a design I've used before.
   [Image #1] Take a look at this image and use it as a reference.
   ```

   :::tip
   اسحب الصور وأفلِتها داخل terminal لإضافتها إلى الطلب.
   :::

   يستطيع Legion فحص أي صور تزوده بها وإضافتها إلى الطلب. يمكنك فعل ذلك عبر سحب الصورة وإفلاتها داخل terminal.

3. **ابنِ الميزة**

   عندما تشعر بالارتياح للخطة، عد إلى _وضع Build_ بالضغط على مفتاح **Tab** مرة أخرى.

   ```bash frame="none"
   <TAB>
   ```

   ثم اطلب منه تنفيذ التغييرات.

   ```bash frame="none"
   Sounds good! Go ahead and make the changes.
   ```

---

### نفّذ تغييرات

للتغييرات الأبسط، يمكنك أن تطلب من Legion تنفيذها مباشرة دون الحاجة لمراجعة خطة أولا.

```txt frame="none" "@packages/functions/src/settings.ts" "@packages/functions/src/notes.ts"
We need to add authentication to the /settings route. Take a look at how this is
handled in the /notes route in @packages/functions/src/notes.ts and implement
the same logic in @packages/functions/src/settings.ts
```

تأكد من تقديم قدر مناسب من التفاصيل لكي يجري Legion التغييرات الصحيحة.

---

### تراجع عن التغييرات

لنفترض أنك طلبت من Legion إجراء بعض التغييرات.

```txt frame="none" "@packages/functions/src/api/index.ts"
Can you refactor the function in @packages/functions/src/api/index.ts?
```

لكن تتضح لك لاحقا أنها ليست ما تريده. يمكنك **التراجع** عن التغييرات باستخدام الأمر `/undo`.

```bash frame="none"
/undo
```

سيقوم Legion الآن بعكس التغييرات التي أُجريت ويعرض رسالتك الأصلية مجددا.

```txt frame="none" "@packages/functions/src/api/index.ts"
Can you refactor the function in @packages/functions/src/api/index.ts?
```

من هنا يمكنك تعديل الطلب وطلب المحاولة مرة أخرى.

:::tip
يمكنك تشغيل `/undo` عدة مرات للتراجع عن عدة تغييرات.
:::

أو يمكنك **إعادة تنفيذ** التغييرات باستخدام الأمر `/redo`.

```bash frame="none"
/redo
```
