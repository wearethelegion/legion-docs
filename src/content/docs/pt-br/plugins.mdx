## Usando um plugin

Existem duas maneiras de carregar plugins.

---

### De arquivos locais

Coloque arquivos JavaScript ou TypeScript no diretﾃｳrio de plugins.

- `.legion/plugins/` - Plugins em nﾃｭvel de projeto
- `~/.config/legion/plugins/` - Plugins globais

Os arquivos nesses diretﾃｳrios sﾃ｣o carregados automaticamente na inicializaﾃｧﾃ｣o.

---

### Do npm

Especifique pacotes npm no seu arquivo de configuraﾃｧﾃ｣o.

```json title="legion.json"
{
  "$schema": "https://wearethelegion.com/config.json",
  "plugin": ["legion-helicone-session", "legion-wakatime", "@my-org/custom-plugin"]
}
```

Pacotes npm regulares e escopados sﾃ｣o suportados.

Navegue pelos plugins disponﾃｭveis no [ecossistema](/docs/ecosystem#plugins).

---

### Como os plugins sﾃ｣o instalados

**Plugins npm** sﾃ｣o instalados automaticamente usando Bun na inicializaﾃｧﾃ｣o. Pacotes e suas dependﾃｪncias sﾃ｣o armazenados em cache em `~/.cache/legion/node_modules/`.

**Plugins locais** sﾃ｣o carregados diretamente do diretﾃｳrio de plugins. Para usar pacotes externos, vocﾃｪ deve criar um `package.json` dentro do seu diretﾃｳrio de configuraﾃｧﾃ｣o (veja [Dependﾃｪncias](#dependencies)), ou publicar o plugin no npm e [adicionﾃ｡-lo ﾃ sua configuraﾃｧﾃ｣o](/docs/config#plugins).

---

### Ordem de carregamento

Os plugins sﾃ｣o carregados de todas as fontes e todos os hooks sﾃ｣o executados em sequﾃｪncia. A ordem de carregamento ﾃｩ:

1. Configuraﾃｧﾃ｣o global (`~/.config/legion/legion.json`)
2. Configuraﾃｧﾃ｣o do projeto (`legion.json`)
3. Diretﾃｳrio de plugins global (`~/.config/legion/plugins/`)
4. Diretﾃｳrio de plugins do projeto (`.legion/plugins/`)

Pacotes npm duplicados com o mesmo nome e versﾃ｣o sﾃ｣o carregados uma vez. No entanto, um plugin local e um plugin npm com nomes semelhantes sﾃ｣o carregados separadamente.

## Criando um plugin

Um plugin ﾃｩ um **mﾃｳdulo JavaScript/TypeScript** que exporta uma ou mais funﾃｧﾃｵes de plugin. Cada funﾃｧﾃ｣o recebe um objeto de contexto e retorna um objeto de hooks.

---

### Dependﾃｪncias

Plugins locais e ferramentas personalizadas podem usar pacotes npm externos. Adicione um `package.json` ao seu diretﾃｳrio de configuraﾃｧﾃ｣o com as dependﾃｪncias necessﾃ｡rias.

```json title=".legion/package.json"
{
  "dependencies": {
    "shescape": "^2.1.0"
  }
}
```

O legion executa `bun install` na inicializaﾃｧﾃ｣o para instalar esses pacotes. Seus plugins e ferramentas podem entﾃ｣o importﾃ｡-los.

```ts title=".legion/plugins/my-plugin.ts"
import { escape } from "shescape"

export const MyPlugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}
```

---

### Estrutura bﾃ｡sica

```js title=".legion/plugins/example.js"
export const MyPlugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}
```

A funﾃｧﾃ｣o do plugin recebe:

- `project`: As informaﾃｧﾃｵes do projeto atual.
- `directory`: O diretﾃｳrio de trabalho atual.
- `worktree`: O caminho do worktree do git.
- `client`: Um cliente SDK do legion para interagir com a AI.
- `$`: A [API shell](https://bun.com/docs/runtime/shell) do Bun para executar comandos.

---

### Enviando notificaﾃｧﾃｵes

Envie notificaﾃｧﾃｵes quando certos eventos ocorrerem:

```js title=".legion/plugins/notification.js"
export const NotificationPlugin = async ({ project, client, $, directory, worktree }) => {
  return {
    event: async ({ event }) => {
      // Send notification on session completion
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Session completed!" with title "legion"'`
      }
    },
  }
}
```

Estamos usando `osascript` para executar AppleScript no macOS. Aqui estamos usando para enviar notificaﾃｧﾃｵes.

:::note
Se vocﾃｪ estiver usando o aplicativo desktop legion, ele pode enviar notificaﾃｧﾃｵes do sistema automaticamente quando uma resposta estiver pronta ou quando ocorrer um erro na sessﾃ｣o.

### Ferramentas personalizadas

Plugins tambﾃｩm podem adicionar ferramentas personalizadas ao legion:

```ts title=".legion/plugins/custom-tools.ts"
import { type Plugin, tool } from "@wearethelegion/plugin"

export const CustomToolsPlugin: Plugin = async (ctx) => {
  return {
    tool: {
      mytool: tool({
        description: "This is a custom tool",
        args: {
          foo: tool.schema.string(),
        },
        async execute(args, context) {
          const { directory, worktree } = context
          return `Hello ${args.foo} from ${directory} (worktree: ${worktree})`
        },
      }),
    },
  }
}
```

O helper `tool` cria uma ferramenta personalizada que o legion pode chamar. Ele aceita uma funﾃｧﾃ｣o de esquema Zod e retorna uma definiﾃｧﾃ｣o de ferramenta com:

- `description`: O que a ferramenta faz
- `args`: Esquema Zod para os argumentos da ferramenta
- `execute`: Funﾃｧﾃ｣o que ﾃｩ executada quando a ferramenta ﾃｩ chamada

Suas ferramentas personalizadas estarﾃ｣o disponﾃｭveis para o legion junto com as ferramentas integradas.

### Hooks de compactaﾃｧﾃ｣o

Personalize o contexto incluﾃｭdo quando uma sessﾃ｣o ﾃｩ compactada:

```ts title=".legion/plugins/compaction.ts"

  return {
    "experimental.session.compacting": async (input, output) => {
      // Inject additional context into the compaction prompt
      output.context.push(`
## Custom Context

Include any state that should persist across compaction:
- Current task status
- Important decisions made
- Files being actively worked on
`)
    },
  }
}
```

O hook `experimental.session.compacting` ﾃｩ acionado antes que o LLM gere um resumo de continuaﾃｧﾃ｣o. Use-o para injetar contexto especﾃｭfico de domﾃｭnio que o prompt de compactaﾃｧﾃ｣o padrﾃ｣o poderia perder.

```ts title=".legion/plugins/custom-compaction.ts"
  return {
    "experimental.session.compacting": async (input, output) => {
      // Replace the entire compaction prompt
      output.prompt = `
You are generating a continuation prompt for a multi-agent swarm session.

Summarize:
1. The current task and its status
2. Which files are being modified and by whom
3. Any blockers or dependencies between agents
4. The next steps to complete the work

`

  }
}
```

Quando `output.prompt` ﾃｩ definido, ele substitui completamente o prompt de compactaﾃｧﾃ｣o padrﾃ｣o. O array `output.context` ﾃｩ ignorado neste caso.
